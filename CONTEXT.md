# 프로젝트 컨텍스트

## 작업 배경
AI 반도체 투자 전략 PDF(Gap and Go, VCP, 3:3:4 피라미딩)를 분석하고,
해당 전략의 통계적 유효성을 검증하기 위해 파이썬 백테스팅 시스템을 구축 중.

## 개발 환경
- Windows 11, Python 3.10, VS Code
- 노트북 ↔ 데스크탑 번갈아 작업 (GitHub으로 동기화)
- 한국 주식 버전은 추후 KIS Open API 연동 예정

### [실험 12] 나스닥100 동적 유니버스 vs 기존 16종목 — 룩어헤드 바이어스 검증
**조건:** 5년, $10,000, exit_mode=hybrid, bear=block MA200, ATR 4%, trailing_stop=on

| 유니버스 | 수익률 | CAGR | MDD | 샤프 | 승률 | 거래수(청산) |
|---------|--------|------|-----|------|------|------------|
| **기존 16종목 (실험10)** | **+590.9%** | **59.1%** | **-21.0%** | **1.72** | 48.6% | 105건 |
| NDX100 동적 유니버스 (97종목) | +144.5% | 23.9% | -22.6% | 0.83 | 36.5% | 208건 |

SPY B&H: +86.9%

**핵심 발견 — 문제 진단:**

1. **룩어헤드 바이어스 수치 확인**: 기존 16종목은 이미 5배+ 오른 걸 알고 고른 cherry-pick 유니버스. 동적 유니버스로 바꾸면 샤프 1.72 → 0.83, 수익률 590% → 144%로 급락. 수치의 상당 부분이 종목 선택 바이어스였음.

2. **LEGACY vs NDX추가 종목 성과 격차**:
   - LEGACY 16종목: 평균 손익 +6.5%, 승률 46.1%
   - NDX100 추가 종목(48개): 평균 손익 -0.3%, 승률 31.9%
   - 추가된 나스닥 종목들이 전략과 궁합이 나쁨 → 잡음 거래 양산

3. **손실 드래그 주범 종목**: APP(-42%), FANG(-24%), MELI(-22%), KLAC(-21%), MRVL(-20%) — 승률 0~33%의 노이즈 종목들
   - 공통점: 고변동성이지만 추세 지속성이 약하거나, 섹터 모멘텀 없이 신호 오발동

4. **청산 사유 분포**: MA10 하이브리드 절반 매도(98건) + MA20 전량(85건)이 대부분 → 청산 로직 자체는 정상 작동. 문제는 진입 품질.

5. **NVDA 진입 10건 승률 40%**: 가장 많이 진입했는데 승률이 낮음 → 신호 점수가 높아도 고점 근처 재진입 반복
